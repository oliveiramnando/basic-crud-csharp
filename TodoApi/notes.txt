creating a new project:
    $ dotnet new webapi --use-controllers -n TodoApi
    $ cd TodoApi
    $ dotnet new webapi --use-controllers -n TodoApi
    $ cd TodoApi

install EntityFrameworkCore
    <sql server provider>
    $ dotnet add package Microsoft.EntityFrameworkCore.SqlServer
    <EF core designe package>
    $ dotnet add package Microsoft.EntityFrameworkCore.Design
    <EF Cli Tool>
    $ dotnet tool update --global dotnet-ef

Run sql locally with Docker; create docker compose file

create databse model 
    - Models/TodoItem.cs
    - use singular names for C# entity classes; each object represents one record

create database context
    - Data/AppDbContext.cs
    - DbContext is EF Core's primary database abstraction
        - opens database connections
        - tracks loaded entities
        - Translates LINQ into SQL 
        - inserts, updates, and deletes records
        - saves changes using SaveChangesAsync()

    - couple of important things here, but the main point is that backend validation should not only exist in the controller
    - Db should also enforce critical rules

configure the connection string
    - appsettings.json
    - Server=localhost,1433
        - connect to sql server on port 1433
    - Database=TodoDb
        - use a database called TodoDb
    - User Id=sa;Password=YourStrong!Password123
        - credentials
    - User Id=sa;Password=YourStrong!Password123
        - accept the development SQL server certificate
    
    - For a real deployed application, do not commit production passwords to appsettings.json. Use environment variables, a secret manager, or platform-managed configuration.

create request and response DTOs (data transfer object)
    - DTOs/Todos
    - do not expose database entity directly from the api. database representation and API representation may need to change independently

    - 3 classes were made cause each represents a different operation
        - create request 
            - client should not decide DB Id, creation time, completion time
        - update request
            - the client can modify title, description, completion status
        - response
            - server returns all public todo information
